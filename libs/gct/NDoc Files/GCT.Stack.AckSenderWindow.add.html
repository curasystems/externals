Adds a new message to the retransmission table. If the message won't have received an ack within a certain time frame, the retransmission thread will retransmit the message to the receiver. If a sliding window protocol is used, we only add up to 

window_size
messages. If the table is full, we add all new messages to a queue. Those will only be added once the table drains below a certain threshold (
min_threshold
