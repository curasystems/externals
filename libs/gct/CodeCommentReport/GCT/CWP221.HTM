GCT.Blocks.DistributedHashtable Class
Provides a simple distributed hashtable. This class behaves identically to the standard system.collections.Hashtable class (e.g. Hashtable[i] = "value" can be used). However, all non-read-only updates will be transmitted to the group. The local hashtables will then be updated on receiving the update message.

Author: Chris Koiak, Bela Ban

Date: 12/03/2003

Access: Public
Base Classes: Hashtable
Implemented interfaces: MessageListener
  Members Description  
    channel The channel that will be used for

  
    ad Used to automatically retrieved messages from the channel

  
    listener Object that will be notified of Add, Remove and Clear operations to the hashtable

  
    state_mutex Used to synchronise initial hashtable state

  
    DistributedHashtable Constructor: Initialises/Connects channel and retreives current hashtable state.

 
    this Gets or Sets the value corresponding to the specified key. Setting a value causes an update on all hashtables.

 
    getState Sends a request for the current state. Method returns once response is received.

 
    Update Updates an entry in the hashtable. Called from "hashtable[key] = value".

 
    Add Adds a value into the Hashtable. This does not update an existing entry.

 
    Clear Clears the hashtable.

 
    Remove Removes the specified key from the hashtable.

 
    Close Closes the associated channel.

 
    receive Message Listener Implementation. Receives messages from the channel and updates the hashtable accordingly.

 
    _Add Called once remote hashtable Add is required.

 
    _Update Called once remote hashtable Update is required.

 
    _Clear Called once remote hashtable Clear is required.

 
    _Remove Called once remote hashtable Remove is required.

 
    _State Called once Set_State message is received. THe current hastable is cleared and the new values inserted.

 
    Listener Interface for objects that required notification of hashtable changes
