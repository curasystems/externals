GCT.Protocols.VERIFY_SUSPECT Class
Protocol: Verifies SUSPECT events traveling up the stack.

Author: Chris Koiak, Bela Ban

Date: 12/03/2003

Access: Public
Base Classes: Protocol
  Members Description  
    local_addr Local Address

  
    timeout Number of millisecs to wait for an i-am-not-dead msg

    suspects Collection of current supsects and time (ms) since suspected

  
    timer Suspects members that havent responded by the timeout

    setProperties Sets the properties specified in the configuration string

 
    up Processes Events travelling up the stack

 
    run Executed in a seperate Thread. Suspects members that haven't responded with i-am-not-dead by the timeout

 
    suspect Sends ARE_YOU_DEAD message to suspected_mbr, wait for return or timeout

 
    unsuspect Unsuspects reponding member

 
    startTimer Starts the timer, run(), to wait for responses

 
    stop Stops the timer, run().

 
Remarks:
Verifies that the suspected member is really dead. If yes, passes SUSPECT event up the stack, otherwise discards it. Has to be placed somewhere above the FD layer and below the GMS layer (receiver of the SUSPECT event).
