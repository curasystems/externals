GCT.Util.Promise Class
Allows a thread to submit an asynchronous request and to wait for the result.

Author: Chris Koiak, Bela Ban

Date: 12/03/2003

Access: Public
Base Classes: Object
  Members Description  
    result The result of the request

  
    mutex Used to wait on the result

  
    getResult If result was already submitted, returns it immediately, else blocks until results becomes available.

 
    checkForResult Checks whether result is available. Does not block.

 
    setResult Sets the result and notifies any threads waiting for it

 
    reset Clears the result and causes all waiting threads to return

 
    toString String representation of the result

 
Remarks:
The caller may choose to check for the result at a later time, or immediately and it may block or not. Both the caller and responder have to know the promise.
